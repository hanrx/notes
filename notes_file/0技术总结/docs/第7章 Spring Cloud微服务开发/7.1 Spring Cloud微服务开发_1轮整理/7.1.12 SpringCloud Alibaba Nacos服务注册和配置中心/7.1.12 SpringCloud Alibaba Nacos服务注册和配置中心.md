# Nacos简介
* 为什么叫Nacos：前四个字母为Nameing和Configuration的前两个字母，最后的s为Service。
* 是什么：
  * 一个更易于构建原生应用的动态服务发现、配置管理和服务管理平台。
  * Nacos：Dynamic Naming and Configuration Service
  * Nacos就是注册中心+配置中心的组合：等价于Nacos=Eureka+Config+bus
* 能干嘛：
  * 替代Eureka做服务注册中心。
  * 替代Config做服务配置中心。
